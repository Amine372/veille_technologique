Projet Java

Java

          L'objectif est de développer un logiciel de gestion de classes. Pour ce faire vous pouvez utiliser le langage JAVA. Votre site doit respecter les standards de JAVA (nom des classes, getter/setter…). Sur votre application, nous devons pouvoir se connecter ainsi que de s’inscrire en tant que professeur. Une fois connecté, le professeur verra toutes les classes dans lequel il enseigne ainsi que son planning. Lorsque l’on clique sur une classe, on peut y mettre les absences et les retards de chaque étudiant d’une classe. Nous pouvons également gérer les sanctions des étudiants (mots dans le carnet, expulsions de cours etc…). Le profil administrateur pourra ajouter des classes ainsi que les étudiants associés aux classes. Il pourra également gérer les étudiants dans les classes. Le professeur principal pourra prendre rendez-vous avec les parents des étudiants. Pour cela, nous pourrons voir un agenda avec les heures pour lequel il sera occupé. Au vu des cours délais, Une répartition des missions par binôme est nécessaire. Chaque binôme doit être autonome et devra fournir un rapport de mission au coordinateur de projet, qui se chargera par la suite de mettre en commun les rapports de chaque binôme dans un seul et unique rapport. Le coordinateur devra veiller à la cohérence du projet ainsi qu’au respect du cahier des charges (les contextes).

          Avec notre site, les élèves renseignent eux-même s'ils veulent manger à la cafétéria un jour donné. De plus, les élèves peuvent programmer les jours pour lesquels ils mangent régulièrement à la cafétéria pour que leurs inscriptions soient automatique. Par défault, si l'élève n'a rien renseigné, on considère évidemment qu'il ne mange aucun jours à la cafétéria. Toutes ces données sont envoyées par courrier éléctronique au cuisinier à 10h sous le format d'un fichier Exel. Le site est concu pour présenter de manière ergonomique les repas que l'élève puisse choisir. Ce projet est le dernier pour lequel nous n'avons pas uttilisés la programmation orientée objet, pour le PHP. Cela a pour but d'affermir nos connaissance en les mettants en application dans un projet concret. La partie PHP a ainsi été le principal volet du développement de ce projet car il exige beaucoup de processus en arrière plan.
